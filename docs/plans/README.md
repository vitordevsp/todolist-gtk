# Plans

Esta pasta guarda frentes de trabalho que precisam sobreviver a mais de uma
sessao.

No recorte do **studio-coding**, planos funcionam como memoria executavel: eles
guardam escopo, tarefas, perguntas e logs para que uma pessoa ou IA consiga
retomar a frente sem depender apenas do chat anterior.

## Estrutura

- [`v1_todolist/`](./v1_todolist/): plano principal da consolidacao da v1.
- [`editorial_articles_from_project_history/`](./editorial_articles_from_project_history/plan.md): frente editorial para transformar a historia real do projeto em reports, candidatos e artigos futuros.
- [`backlog.md`](./backlog.md): ideias futuras fora do fechamento atual.
- [`todolist.md`](./todolist.md): inbox simples para notas soltas antes de virarem plano ou backlog.

## Plano Como Pasta

Quando uma frente crescer, prefira uma pasta propria com:

- `plan.md`: objetivo, contexto e criterio de aceite.
- `tasks.md`: lista de tarefas e status.
- `logs.md`: historico cronologico de decisoes e validacoes.
- `questions.md`: perguntas em aberto para decisao humana, quando necessario.

Use [`../patterns/plan-patterns.md`](../patterns/plan-patterns.md) como regra base.
