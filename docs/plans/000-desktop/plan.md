# Plano 000: Desktop de Planos

## Objetivo

Manter uma area operacional leve para notas soltas e backlog geral antes que
elas virem uma frente numerada propria.

Este plano funciona como entrada do sistema de planos:

- `todolist.md`: inbox para notas, ideias e lembretes ainda sem destino claro;
- `backlog.md`: ideias futuras que ja fazem sentido, mas ainda nao justificam
  um plano dedicado.

## Regra de Uso

- Use `todolist.md` para capturar rapido.
- Mova para `backlog.md` quando a ideia ja for uma melhoria futura reconhecida.
- Crie um plano numerado quando a frente precisar de contexto, tarefas, logs ou
  perguntas proprias.

## Escopo

Inclui:

- triagem inicial de ideias;
- backlog geral do projeto;
- ponte entre conversa solta e plano estruturado.

Nao inclui:

- execucao de tarefas complexas;
- logs detalhados de implementacao;
- fonte de verdade de produto ou arquitetura.
