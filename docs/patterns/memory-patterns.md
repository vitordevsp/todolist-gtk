# Padrão: Registro de Memória (MEMORY.md)

Este documento define como os aprendizados técnicos e decisões arquiteturais devem ser registrados para garantir a continuidade do projeto.

## Estrutura do Arquivo MEMORY.md
O arquivo de memória na raiz do projeto deve seguir esta hierarquia:

1.  **🧠 Desafios e Soluções (Brain):** Registro de problemas técnicos complexos.
    - **Formato:** Problema -> Contexto -> Solução técnica adotada.
2.  **🛠️ Tecnologias Validadas:** Lista de bibliotecas, ferramentas ou padrões que provaram ser eficazes no projeto.
3.  **⚠️ Armadilhas (Pitfalls):** Coisas que tentamos e não funcionaram, ou bugs específicos de versão que exigem atenção.

## Relação com o Backlog
Diferente da Memória, que registra o **passado** (o que aprendemos), o arquivo `backlog.md` dentro de uma pasta de plano (`docs/plans/*/`) registra o **futuro** (o que queremos fazer mas não faremos agora).
