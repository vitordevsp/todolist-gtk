# Patterns

Esta pasta guarda padroes pequenos usados para manter a documentacao e a memoria
do projeto consistentes.

No TodoList GTK ela ainda e propositalmente minima. A ideia e testar uma pequena
semente do framework **studio-coding**, nao copiar toda a estrutura dele.

## Padroes Atuais

- [`memory-patterns.md`](./memory-patterns.md): como registrar aprendizados no `MEMORY.md`.
- [`plan-patterns.md`](./plan-patterns.md): como organizar planos como pasta.

## Quando Criar Um Novo Pattern

Crie um novo pattern apenas quando uma regra ou convencao for reutilizada mais de
uma vez.

Nao use `patterns/` para registrar decisoes pontuais de implementacao. Nesse
caso, prefira o `logs.md` do plano ativo ou um guia em `docs/knowledge/`.
