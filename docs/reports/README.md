# Reports

Esta pasta guarda relatorios intermediarios usados para investigar, sintetizar e
preparar material editorial.

No framework `studio-coding`, reports sao diferentes de artigos:

- reports organizam sinais, evidencias, lacunas e interpretacoes;
- articles transformam um recorte aprovado em texto publico.

## Relatorios Desta Frente

- [`project-timeline/`](./project-timeline/README.md): linha do tempo do projeto.
- [`technical-decisions/`](./technical-decisions/README.md): decisoes tecnicas e pontos de virada.
- [`docs-framework-evolution/`](./docs-framework-evolution/README.md): evolucao da camada `docs/` como mini-framework.
- [`memory-and-learning-signals/`](./memory-and-learning-signals/README.md): aprendizados extraidos de memoria, changelog e logs.
- [`conversation-sources/`](./conversation-sources/README.md): conversas relevantes e como usa-las sem perder contexto.
- [`editorial-themes/`](./editorial-themes/README.md): temas editoriais emergentes.
- [`article-candidates/`](./article-candidates/README.md): relatorio final de candidatos a artigos.

## Criterios de Uso

Um sinal merece virar report quando:

- ajuda a explicar a historia real do projeto;
- precisa ser cruzado com outras fontes;
- ainda nao esta pronto para virar artigo;
- tem ambiguidade que deve ficar visivel para sessoes futuras.

Um sinal deve continuar apenas como nota interna quando:

- for detalhe operacional sem valor narrativo;
- depender de confirmacao que ainda nao existe;
- duplicar uma fonte melhor ja registrada em plano, memoria ou changelog.
