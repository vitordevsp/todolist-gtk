TodoList GTK é pequeno por escolha. O que importa aqui é o ciclo: sair de uma ideia antiga, entrar numa stack desconhecida, usar IA como parceira e transformar isso em um app real, instalável e documentado.

Sobre o projeto

Isso começou como um teste pessoal para entrar numa stack nova sem parar a vida.

Eu queria construir um app nativo para Linux fazia tempo, mas sempre esbarrava no mesmo custo: aprender GTK, Python e o ecossistema desktop com a profundidade que isso pede.

Esse projeto nasceu no meio do caminho, não em condições perfeitas. Algumas horas aqui, outras ali. Direção, iteração, revisão. Foi mais parecido com pair programming real do que com delegação cega.

O resultado foi um app funcional e, junto dele, uma forma mais honesta de pensar construção com IA.

Por que isso existe

Um laboratório para aprender Linux de verdade, com software de verdade.

Aprender fazendo

Em vez de consumir semanas estudando abstratamente a stack, o projeto usou um app real como trilha de entrada.

Testar processo

O experimento não era só técnico. Era entender como IA, docs e especificação mudam o fluxo de desenvolvimento.

Ganhar repertório

O produto

Simples por escolha. Completo o bastante para fechar um ciclo real.

Múltiplas listas

Estrutura de barra lateral com sessões para separar trabalho, estudos, ideias e qualquer outro contexto.

CRUD de tarefas

Criação, edição, renomeação e exclusão dentro de um fluxo direto, sem ruído de produto inflado.

Persistência local

Dados guardados em SQLite local, respeitando o ambiente Linux e sem dependências de conta ou nuvem.

Live reload

Workflow rápido para iterar interface sem transformar cada ajuste em custo desnecessário.

Build, bundle e cara de app nativo

O projeto não parou na UI. Passou por build local, Flatpak e distribuição como app Linux de verdade, com a intenção de parecer pertencer ao desktop.

Stack real

Entrar numa stack nova deixou de ser bloqueio absoluto.

O app foi construído com Python, GTK4, Libadwaita, SQLite e Flatpak. Nada disso era minha stack principal. Justamente por isso o experimento importa.

Python + PyGObject

Base da aplicação desktop e ponte com GTK/Libadwaita.

GTK4 + Libadwaita

Interface nativa, componentes modernos e integração melhor com Linux.

SQLite

Persistência local simples, direta e suficiente para o escopo.

Flatpak

Empacotamento e distribuição coerentes com a proposta de app Linux.

Laboratório

O valor desse projeto não está só no app. Está no jeito de construir.

Esse projeto começou como um teste. Eu queria entender até onde dava pra ir usando IA como parceira. No meio do caminho ficou claro: isso não era só sobre um app. Era sobre aprender a construir de um jeito diferente.

O Studio Coding entrou como framework de trabalho: plano para dar direção, specs para transformar intenção em contrato, logs para não perder contexto, padrões para repetir o que funcionou e memória para atravessar sessões longas sem recomeçar do zero.

A taxonomia importa porque tira a IA do improviso. Projeto, plano, tarefa, decisão, rotina, skill e documentação deixam de ser arquivos soltos e viram um mapa de desenvolvimento. O ganho não é automação mágica. É direção humana com um sistema que ajuda a IA a trabalhar melhor.
